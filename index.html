🧠 What is Two Pointer?

Two Pointer is a technique where you use two index variables to traverse a data structure (usually an array or string) simultaneously — instead of using nested loops.

💡 Core Insight: When a problem involves searching pairs, subarrays, or comparisons in a linear structure — two pointers can reduce O(n²) to O(n).

Think of it like two people reading a book — one from the front, one from the back — and they walk toward each other until they meet.

⚡ Why Does It Work?
BRUTE FORCE
O(n²)
Check every pair — 2 nested loops
TWO POINTER
O(n)
Smart convergence — single pass
🔑 When to Use Two Pointers?

Ask yourself these questions when you see a problem:

✓
Sorted array? Two pointers shine on sorted data — you can decide which pointer to move based on comparisons.
✓
Find a pair/triplet with sum = target? Classic two-sum pattern.
✓
Palindrome / reverse check? One from start, one from end.
✓
Remove duplicates / partition array? Slow & fast pointer pattern.
✓
Linked list cycle / middle? Fast & slow pointer (Floyd's algorithm).
🗺️ The 3 Flavors of Two Pointers
OPPOSITE ENDS
Converging Pointers
L starts at index 0, R at end. They move toward each other. Used for: pair sum, palindrome, container with most water.
SAME DIRECTION
Fast & Slow
Both start at 0. Slow writes, fast reads ahead. Used for: remove duplicates, partition, linked list cycle.
TWO ARRAYS
Parallel Traversal
One pointer per array. Used for: merge sorted arrays, compare strings, intersection.
🎯 Pattern 1: Opposite Ends (Two Sum Sorted)

Array is sorted. Find two numbers that sum to a target.

Logic: If sum < target → move L right (need bigger). If sum > target → move R left (need smaller). If sum = target → found! 
def two_sum_sorted(arr, target):
    L, R = 0, len(arr) - 1
    
    while L < R:
        s = arr[L] + arr[R]
        
        if s == target:
            return [L, R]          # ✅ Found!
        elif s < target:
            L += 1                 # need bigger → move L right
        else:
            R -= 1                 # need smaller → move R left
    
    return []                   # not found
⏱ Time: O(n)
💾 Space: O(1)
🔄 Pattern 2: Fast & Slow — Remove Duplicates

Sorted array — remove duplicates in-place.

Logic: slow = write position. fast = reader. Only write when fast finds a new value. 
def remove_duplicates(nums):
    if not nums: return 0
    
    slow = 0               # slow = last written position
    
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            slow += 1       # move write head forward
            nums[slow] = nums[fast]  # write unique value
    
    return slow + 1          # count of unique elements

# [1,1,2,3,3,4] → [1,2,3,4,_,_], returns 4
🏃 Pattern 3: Palindrome Check
def is_palindrome(s):
    L, R = 0, len(s) - 1
    
    while L < R:
        if s[L] != s[R]:
            return False      # mismatch found
        L += 1
        R -= 1
    
    return True

# "racecar" → True | "hello" → False
🧩 Pattern 4: Three Sum (Reduces to Two Sum)
Key Insight: Sort first. Fix one element, then use two pointers on the rest. O(n²) total — much better than O(n³) brute force. 
def three_sum(nums):
    nums.sort()
    result = []
    
    for i in range(len(nums) - 2):
        # Skip duplicates for fixed element
        if i > 0 and nums[i] == nums[i-1]: continue
        
        L, R = i + 1, len(nums) - 1
        
        while L < R:
            s = nums[i] + nums[L] + nums[R]
            if s == 0:
                result.append([nums[i], nums[L], nums[R]])
                while L < R and nums[L] == nums[L+1]: L += 1
                while L < R and nums[R] == nums[R-1]: R -= 1
                L += 1; R -= 1
            elif s < 0: L += 1
            else: R -= 1
    
    return result

EASY Valid Palindrome (LeetCode 125) ▼

Check if a string is a palindrome (only alphanumeric characters, case-insensitive).

def isPalindrome(s):
    L, R = 0, len(s) - 1
    while L < R:
        # Skip non-alphanumeric
        while L < R and not s[L].isalnum(): L += 1
        while L < R and not s[R].isalnum(): R -= 1
        if s[L].lower() != s[R].lower(): return False
        L += 1; R -= 1
    return True
💡 Tip: Skip chars that aren't a-z or 0-9 before comparing. Inner while loops handle skipping efficiently.
EASY Move Zeroes to End (LeetCode 283) ▼

Move all 0s to end while maintaining relative order of non-zero elements.

def moveZeroes(nums):
    slow = 0   # write position
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow] = nums[fast]
            slow += 1
    # Fill rest with zeros
    while slow < len(nums):
        nums[slow] = 0
        slow += 1

# [0,1,0,3,12] → [1,3,12,0,0]
MEDIUM Container With Most Water (LeetCode 11) ▼

Find two lines that form a container holding the most water.

💡 Intuition: Area = min(height[L], height[R]) × (R - L). Always move the shorter side inward — keeping the taller side can only help.
def maxArea(height):
    L, R = 0, len(height) - 1
    max_water = 0
    
    while L < R:
        h = min(height[L], height[R])
        water = h * (R - L)
        max_water = max(max_water, water)
        
        # Move the shorter wall
        if height[L] < height[R]:
            L += 1
        else:
            R -= 1
    
    return max_water
MEDIUM Trapping Rain Water (LeetCode 42) ▼

Given elevation map, compute how much water it can trap.

💡 Key: Water at any position = min(max_left, max_right) - height[i]. Two pointers track running max from each side.
def trap(height):
    L, R = 0, len(height) - 1
    left_max = right_max = water = 0
    
    while L < R:
        if height[L] < height[R]:
            if height[L] >= left_max:
                left_max = height[L]
            else:
                water += left_max - height[L]
            L += 1
        else:
            if height[R] >= right_max:
                right_max = height[R]
            else:
                water += right_max - height[R]
            R -= 1
    
    return water
HARD Minimum Window Substring (LeetCode 76) ▼

Find minimum window in s containing all chars of t. Uses sliding window variant of two pointers.

from collections import Counter

def minWindow(s, t):
    need = Counter(t)
    have, total = 0, len(need)
    window = {}
    res, resLen = [-1, -1], float('inf')
    L = 0
    
    for R in range(len(s)):
        c = s[R]
        window[c] = window.get(c, 0) + 1
        if c in need and window[c] == need[c]:
            have += 1
        
        while have == total:
            if (R - L + 1) < resLen:
                res, resLen = [L, R], R - L + 1
            window[s[L]] -= 1
            if s[L] in need and window[s[L]] < need[s[L]]:
                have -= 1
            L += 1
    
    L, R = res
    return s[L:R+1] if resLen != float('inf') else ""
🚀 Advanced: Linked List — Cycle Detection (Floyd's)
Slow & Fast on Linked Lists: Slow moves 1 step, Fast moves 2 steps. If there's a cycle, they'll eventually meet. If fast reaches null — no cycle. 
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next           # 1 step
        fast = fast.next.next      # 2 steps
        if slow == fast:
            return True           # cycle found!
    return False

def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # slow is at middle!
🔀 Advanced: Merge Two Sorted Arrays
Two arrays, two pointers. Each pointer advances in its own array. Pick the smaller element each time. 
def merge_sorted(A, B):
    i = j = 0
    result = []
    
    while i < len(A) and j < len(B):
        if A[i] <= B[j]:
            result.append(A[i]); i += 1
        else:
            result.append(B[j]); j += 1
    
    result += A[i:] + B[j:]  # append remaining
    return result

⚠️ Common Mistakes:

1. Not sorting first — Two pointer on opposite ends only works on sorted arrays for pair problems.

2. Off-by-one — Use while L < R (not ≤) to avoid processing the same element twice.

3. Forgetting duplicate skipping in 3Sum — without it, you get repeated triplets.

4. Wrong pointer to move — always move based on the comparison result logically.

📋 Two Pointer Cheat Sheet
OPPOSITE ENDS
L=0, R=n-1. Move based on comparison. Array must be sorted for sum problems. Loop: while L < R
FAST & SLOW
Slow = write head. Fast = read head. Advance slow only when condition met. Used for in-place operations.
SLIDING WINDOW
Expand R to add elements, shrink L when constraint violated. Track window state with hashmap/counter.
FLOYD'S ALGO
slow += 1, fast += 2. Meet = cycle exists. Find middle: fast reaches end when slow is at mid.
3SUM TRICK
Sort → Fix i → Two pointer on [i+1 .. n-1]. Skip duplicates with: if nums[i] == nums[i-1]: continue
RAIN WATER
Track left_max and right_max. Move pointer on smaller side. Water = max - current height.
CONTAINER WATER
Area = min(h[L],h[R]) × (R-L). Always move the shorter side inward to potentially find a taller wall.
